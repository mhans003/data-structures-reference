Binary Heaps

  • Type of heap with at most two children
  • Heaps require children/parents to always be greater/less than each other, depending on type.
  • Unlike binary search trees, there is no order to left vs. right values, nor between siblings (only parent->child)
  • Compact as possible - All left/right are always filled out first before going downward.
  • Left is prioritized first to be filled.
  • Max Binary Heap: Parent nodes are always greater than child nodes
  • Min Binary Heap: Parent nodes are always less than child nodes

Binary Heaps as Arrays

  • For any given index (parent node) n, the left child is located at 2n + 1; the right child is at 2n + 2.
  • For any given index (child node) n, the parent is located at (n - 1)/2 (floored/rounded down).
