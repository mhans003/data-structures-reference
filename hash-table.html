Hash Tables

Used to store key-value pairs.

Hash Tables:

  • Data structure that provides/maps to key-value pairs
  • Uses hash function to convert keys to a valid array index.
    • Should be fast
    • Should produce same hash with same input
  • Actual values stored in array can contain other data structures, such as linked lists.
    • Data structure at desired index can contain original key alongside value.
    • As a result, mutiple items can be stored at one index (separate chaining)
    • This can handle 'collisions' where two keys/indexes can be converted to same array index.
    • Alternatively, linear probing can be used to keep one item at each slot
  • Finding, adding, and removing values are quick
  • Examples (built-in structures) in various languages:
    • Python - Dictionaries
    • JS - Objects/Maps
    • JS Object keys are converted to strings
    • Java/Go/Scala - Maps
    • Ruby - Hashes
  • Many applications in security and cryptography

Big O

  • Insertion - O(1)
  • Removal - O(1)
  • Access - O(1)
