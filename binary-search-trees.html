Binary Search Trees

  • Binary trees have nodes with at most two children each
  • Binary search trees have nodes on left with keys smaller than root key
  • Binary search trees have nodes on right with keys larger than root key

trees:

  • Nodes containing pointers to child node(s)
  • Non-linear (unlike lists)
  • Nodes can have no children, a single child, or multiple children.
  • Only parent->child relationship; Cannot point to parent/sibling nodes (unlike graphs)
  • Trees need to have one root (root node).
  • "Leaves" are nodes with no children.
  • Edge - Connection between one node and another
  • Examples include:
    • HTML DOM
    • Network routing
    • Abstract syntax trees (for programming language abstractions)
    • Artificial Intelligence
    • OS folders and computer file systems
    • JSON

Big O

  • Insertion - O(log n)
  • Searching - O(log n)
